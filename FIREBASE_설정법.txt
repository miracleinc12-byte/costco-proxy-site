Firebase (구글 클라우드 DB) 주문 저장 — 설정 가이드
══════════════════════════════════════════════════

주문이 들어오면 구글 Firestore DB에 자동 저장됩니다.
설정 전까지는 기존처럼 이메일 알림만 작동합니다 (사이트는 정상 동작).
전체 소요시간: 약 10분. 완전 무료입니다 (하루 주문 2만 건까지).

────────────────────────────────────────────────
STEP 1. Firebase 프로젝트 만들기
────────────────────────────────────────────────
1. https://console.firebase.google.com 접속 (구글 계정 로그인)
2. "프로젝트 추가" 클릭
3. 프로젝트 이름: costco-proxy (아무거나 가능)
4. 구글 애널리틱스: "사용 안함" 선택 → 만들기

────────────────────────────────────────────────
STEP 2. Firestore 데이터베이스 만들기
────────────────────────────────────────────────
1. 왼쪽 메뉴 → 빌드 → Firestore Database → "데이터베이스 만들기"
2. 위치: asia-northeast3 (서울) 선택
3. "프로덕션 모드로 시작" 선택 → 만들기

────────────────────────────────────────────────
STEP 3. 보안 규칙 설정 (중요!)
────────────────────────────────────────────────
Firestore → 규칙 탭 → 아래 내용으로 교체 → "게시"

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /orders/{orderId} {
      // 누구나 주문 생성만 가능. 조회·수정·삭제는 관리자(콘솔)만.
      allow create: if request.resource.data.keys().hasAll(['orderNo','items','name','phone'])
                    && request.resource.data.items is string
                    && request.resource.data.items.size() < 2000;
      allow read, update, delete: if false;
    }
  }
}

★ 이 규칙 덕분에: 방문자는 주문 넣기만 가능하고,
  다른 사람의 주문(개인정보)을 절대 볼 수 없습니다.

────────────────────────────────────────────────
STEP 4. 웹 앱 등록 + 설정값 복사
────────────────────────────────────────────────
1. 프로젝트 개요(홈) → "</>" (웹) 아이콘 클릭
2. 앱 닉네임: costco-site → "앱 등록"
3. 화면에 나오는 firebaseConfig = { ... } 부분을 복사

────────────────────────────────────────────────
STEP 5. order.html에 붙여넣기
────────────────────────────────────────────────
order.html 파일을 열어 아래 부분을 찾아서:

  const firebaseConfig = {
    apiKey: "YOUR_API_KEY",
    ...
  };

STEP 4에서 복사한 실제 값으로 교체하고 저장.
→ 끝! 이제 주문이 DB에 자동 저장됩니다.

────────────────────────────────────────────────
주문 확인하는 법
────────────────────────────────────────────────
· Firebase 콘솔 → Firestore Database → orders 컬렉션
· 주문마다: 주문번호, 상품, 금액, 이름, 연락처, 주소, 상태(입금대기)
· 입금 확인하면 콘솔에서 status를 "입금완료" → "발송완료"로 직접 수정 가능
· 휴대폰에서도 console.firebase.google.com 접속하면 확인 가능

────────────────────────────────────────────────
동작 방식
────────────────────────────────────────────────
주문 완료 버튼 클릭
  → ① Firestore DB에 저장 (주문번호 자동 생성: 20260708-A3F2 형식)
  → ② formsubmit.co로 이메일 알림 발송 (기존 기능, 백업용)
  → ③ 주문 완료 페이지에 주문번호 표시
※ DB 저장이 실패해도 이메일은 무조건 발송되어 주문이 유실되지 않습니다.

────────────────────────────────────────────────
STEP 6. 관리자 페이지 (admin.html) 활성화
────────────────────────────────────────────────
admin.html = 주문 목록·입금확인·운송장 입력을 한 화면에서 처리하는
사장님 전용 페이지입니다. 활성화하려면:

6-1. 구글 로그인 켜기
  1. Firebase 콘솔 → 빌드 → Authentication → "시작하기"
  2. 로그인 방법 탭 → "Google" 선택 → 사용 설정 토글 ON
  3. 지원 이메일: miracleinc12@gmail.com 선택 → 저장

6-2. 보안 규칙 업데이트 (관리자만 주문 조회 가능하게)
  Firestore → 규칙 탭 → 아래 내용으로 전체 교체 → 게시

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /orders/{orderId} {
      allow create: if request.resource.data.keys().hasAll(['orderNo','items','name','phone'])
                    && request.resource.data.items is string
                    && request.resource.data.items.size() < 2000;
      allow read, update, delete: if request.auth != null
                    && request.auth.token.email == 'miracleinc12@gmail.com'
                    && request.auth.token.email_verified == true;
    }
  }
}

★ miracleinc12@gmail.com 구글 계정으로 로그인한 사람만
  주문을 보고 수정할 수 있습니다. 다른 계정은 전부 차단됩니다.

6-3. 사용법
  · admin.html 접속 → "구글 계정으로 로그인" 클릭
  · 주문이 최신순으로 나오고, 상태 칩(입금대기/발송완료 등)으로 필터
  · 운송장 번호 입력 + 상태 변경 → 💾 저장
  · "📋 고객 안내문 복사" → 카카오톡에 붙여넣어 고객에게 전송
    (상태에 따라 입금 안내/입금 확인/발송 안내 문구가 자동 선택됨)

────────────────────────────────────────────────
개인정보 주의사항
────────────────────────────────────────────────
주문 DB에는 고객 이름·연락처·주소가 저장됩니다.
· 보안 규칙(STEP 3)을 반드시 적용하세요 (외부 조회 차단)
· 배송 완료된 지 오래된 주문은 주기적으로 삭제하는 것을 권장합니다
